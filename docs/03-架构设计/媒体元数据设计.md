# 封面歌词自动补全方案设计

## 1. 音频元数据读写方案

### 1.1 MP3（ID3v2）标签读写

| 库 | 平台 | 说明 |
|----|------|------|
| `flutter_id3_reader` | Flutter | 读取 MP3 ID3v2 标签，支持封面/歌词提取 |
| `id3_codec` | Dart | 读写 ID3v1/v2 标签 |
| `dart_id3_tag_editor` | Dart | 写入 ID3v2 标签 |

**推荐**：`flutter_id3_reader` 读取 + 自研轻量写入（ID3v2 规范相对简单，可直接拼接字节）

**ID3v2 帧参考：**
| 帧 | 说明 | 对应数据 |
|----|------|----------|
| APIC | 封面图片 | JPEG/PNG 数据 |
| SYLT | 同步歌词 | 时间戳 + 歌词内容 |
| USLT | 非同步歌词 | 纯文本歌词 |

### 1.2 MP4/AAC（M4A）元数据

| 方案 | 说明 |
|------|------|
| `mp4_data` | Dart 解析 MP4 box 结构 |
| Platform Channel + 原生 | Android `MediaMetadataRetriever`，iOS `AVAsset` |

**推荐**：通过 Platform Channel 调用原生接口，确保支持完整 iTunes 规范。

### 1.3 文件安全策略

> **先备份再写入**：任何元数据写入前，先将原文件复制为 `.bak` 文件。

```
写入流程：
[读取原文件字节]
    ↓
[复制为 *.mp3.bak]
    ↓
[修改元数据后写回原路径]
    ↓
[验证：可读取且不破坏音频数据]
    ↓ (失败)
[删除损坏文件，恢复 .bak]
```

## 2. 封面搜索方案

### 2.1 数据源对比

| API | 稳定性 | 覆盖度 | 认证方式 | 推荐指数 |
|-----|--------|--------|----------|----------|
| QQ 音乐封面 | 高 | 流行音乐 | 无需认证（公开接口） | ⭐⭐⭐⭐ |
| 网易云音乐封面 | 中 | 广 | 需要 Key（部分开放） | ⭐⭐⭐ |
| 酷狗封面 | 中 | 一般 | 无需认证 | ⭐⭐ |
| Last.fm | 中 | 一般 | API Key | ⭐⭐ |

**推荐**：优先 QQ 音乐封面 API，兜底使用 Last.fm。

### 2.2 QQ 音乐封面搜索

```
接口（公开，非官方）：
GET https://c.y.qq.com/soso/fcgi-bin/client_search_cp?
    w={song_name}+{artist_name}&format=json

响应中提取：专辑图片地址（通常 300x300 ~ 800x800）
```

### 2.3 搜索匹配流程

```
[提取歌曲名 + 歌手名（已有 metadata）]
    ↓
[拼接查询词，向 QQ 音乐 API 发起请求]
    ↓
[解析响应，获取专辑封面 URL 列表]
    ↓
[按文件名、大小筛选最合适的一张]
    ↓ (匹配度 > 80%)
[下载图片到临时目录]
    ↓
[提取歌曲 hash 或生成唯一 ID]
    ↓
[缓存封面到本地 CoverCache/{hash}.jpg]
    ↓
[嵌入 MP3 APIC 帧 / MP4 moov.udta.meta]
```

### 2.4 图片格式处理

| 格式 | 处理 |
|------|------|
| JPG | 直接嵌入 APIC，MIME type = `image/jpeg` |
| PNG | 转换为 JPG 后嵌入（或保持 PNG，MIME = `image/png`）|
| WebP | 转换为 JPG，兼容性更佳 |

> 建议统一转换为 500x500 以内的 JPG，减少文件体积膨胀。

## 3. 歌词搜索方案

### 3.1 LRC 文件格式

LRC 是基于文本的歌词格式，核心规范：

```
[ti:歌曲名]
[ar:歌手]
[al:专辑]
[offset:0]

[00:12.34]第一行歌词
[00:15.67]第二行歌词
```

**时间戳正则**：`\[(\d{2}):(\d{2})\.(\d{2})\]`

### 3.2 数据源

| API | 说明 |
|-----|------|
| QQ 音乐歌词 | 接口需签名，推荐用公开抓取方案 |
| 网易云音乐歌词 | 接口较稳定，部分需模拟 UA |
| lrc歌词网 | lrcnet.com 等老站，资源较全 |
| LRCDog | 公开 LRC 搜索引擎 |

**兜底策略**：主选 QQ 音乐 + 网易云，失败后搜索 LRCDog。

### 3.3 搜索流程

```
[获取歌曲名 + 歌手名]
    ↓
[向 QQ 音乐 / 网易云 发送搜索请求]
    ↓ (失败则换数据源)
[解析响应，提取 LRC 文本]
    ↓
[校验：是否包含时间戳，是否有超过 5 句歌词]
    ↓ (有效)
[下载 LRC 到临时目录]
    ↓
[存入 CoverCache/{hash}.lrc]
    ↓
[方案A：嵌入文件内部] → 写入 ID3 SYLT/USLT 帧
[方案B：保存同名字幕文件] → 存为 song_name.lrc 与音频同目录
```

> **推荐方案 B**：保存为 `.lrc` 同名文件，不破坏原始音频，兼容性好，用户可导出。

### 3.4 嵌入文件内部（方案A）实现

```dart
// MP3 SYLT 帧结构（简化）
// 帧头：'SYLT' + size(4bytes) + flags(2bytes)
// 内容：< BOM + language + time_format + content_type + {timestamp}{lyric}\n }
// 时间格式：2 = 毫秒（ms）
```

## 4. 匹配准确率

### 4.1 封面匹配

| 指标 | 目标 |
|------|------|
| 封面匹配成功率 | > 80% |
| 误匹配率（封面与歌曲不符） | < 5% |

**准确率提升策略：**
- 歌曲名 + 歌手名双重匹配
- 匹配前标准化处理（全角转半角、去除特殊字符）
- 多结果时取专辑封面（而非歌手写真），准确率更高

### 4.2 歌词匹配

| 指标 | 目标 |
|------|------|
| 歌词匹配成功率 | > 80% |
| 时间轴准确率 | > 70% |

**匹配失败降级：**
- 显示"暂无歌词"占位
- 记录失败 song_id 和时间，供后续补充
- 提供用户手动搜索入口

## 5. 实现流程

### 5.1 整体流程图

```
[文件扫描完成，生成 song_metadata]
    ↓
[检测是否有封面] ── 无 ──→ [搜索封面 API]
    ↓ (有)                          ↓
[检测是否有歌词] ── 无 ──→ [搜索歌词 API]
    ↓ (有)                          ↓
        ↓                           ↓
    [下载资源] ←────────────────────┘
    ↓
[转码 / 格式化]
    ↓
[方案B: 保存为 .lrc 文件]
[方案A: 嵌入元数据（可选）]
    ↓
[写入本地 CoverCache]
    ↓
[更新 song_metadata 表，标记封面/歌词已补全]
    ↓
[前端刷新 UI，显示新封面/歌词]
```

### 5.2 性能优化

| 策略 | 说明 |
|------|------|
| 异步搜索 | 补全在后台执行，不阻塞播放 |
| 批量处理 | 每批 10 首，避免请求过于密集 |
| 缓存命中 | 已补全的歌曲跳过重复搜索 |
| 离线队列 | 无网络时将补全任务入队，恢复后执行 |
| 限速 | API 请求间隔 ≥ 500ms，避免触发限流 |

### 5.3 用户通知

- 补全成功：静默更新，用户切到歌曲页时自然看到新封面/歌词
- 补全失败：记录日志，可在"音乐管理"页展示"有 N 首缺失封面/歌词"，用户可手动触发补全
