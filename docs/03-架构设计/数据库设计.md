# Vexfy 数据库设计文档

> 本文档定义 SQLite 本地数据库的表结构设计。
> 数据库文件路径由 `path_provider` 动态获取，应用私有目录下的 `vexfy.db`。

---

## 1. 数据库概览

- **数据库引擎**：SQLite 3（通过 `sqflite` 包访问）
- **数据库文件**：`getApplicationDocumentsDirectory()/vexfy.db`
- **版本**：v1（初始版本）
- **字符编码**：UTF-8（SQLite 默认）

---

## 2. 表总览

| 表名 | 说明 | 主键 |
|------|------|------|
| `playlists` | 歌单表 | `id` |
| `playlist_songs` | 歌单-歌曲关联表 | `id` (自增) |
| `song_stats` | 歌曲播放统计表 | `song_id` |
| `played_history` | 播放历史表 | `id` (自增) |
| `local_file_metadata` | 本地音乐元数据表 | `song_id` |
| `sync_queue` | OSS 同步队列表 | `id` (自增) |

---

## 3. 表结构详情

---

### 3.1 playlists — 歌单表

存储用户创建的歌单信息。

```sql
CREATE TABLE playlists (
  id          TEXT PRIMARY KEY,    -- 歌单ID，UUID 或哈希
  name        TEXT NOT NULL,       -- 歌单名称
  cover_url   TEXT,                -- 封面图URL
  description TEXT,                -- 歌单描述
  creator     TEXT NOT NULL,       -- 创建者标识
  type        TEXT NOT NULL,       -- 类型：userCreated / onlineImported / favorite / history
  song_count  INTEGER DEFAULT 0,   -- 歌曲数量（冗余字段，加速查询）
  created_at  TEXT NOT NULL,       -- 创建时间，ISO 8601 字符串
  updated_at  TEXT NOT NULL        -- 最后修改时间，ISO 8601 字符串
);
```

**索引设计**：

```sql
-- 按创建时间倒序查询索引
CREATE INDEX idx_playlists_created_at ON playlists(created_at DESC);

-- 按类型查询索引
CREATE INDEX idx_playlists_type ON playlists(type);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `TEXT` | PK | 歌单唯一标识 |
| `name` | `TEXT` | NOT NULL | 歌单名称，最大 200 字符 |
| `cover_url` | `TEXT` | 可空 | 封面图 URL |
| `description` | `TEXT` | 可空 | 歌单描述，最大 500 字符 |
| `creator` | `TEXT` | NOT NULL | 创建者标识（如 user_001） |
| `type` | `TEXT` | NOT NULL | 歌单类型枚举字符串 |
| `song_count` | `INTEGER` | DEFAULT 0 | 歌曲数量（变更时同步更新） |
| `created_at` | `TEXT` | NOT NULL | 创建时间，UTC |
| `updated_at` | `TEXT` | NOT NULL | 最后修改时间，UTC |

### 3.2 playlist_songs — 歌单歌曲关联表

存储歌单与歌曲的关联关系，支持同一首歌在多个歌单中出现。

```sql
CREATE TABLE playlist_songs (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增主键
  playlist_id TEXT NOT NULL,                      -- 歌单ID，外键
  song_id     TEXT NOT NULL,                      -- 歌曲ID（本地为路径哈希，在线为平台ID）
  song_source TEXT NOT NULL,                      -- 歌曲来源：local / online
  title       TEXT NOT NULL,                      -- 歌曲标题（冗余存储，避免关联查询）
  artist      TEXT NOT NULL,                      -- 歌手（冗余存储）
  duration    INTEGER,                            -- 时长，毫秒（冗余存储）
  file_path   TEXT,                               -- 本地路径（仅本地歌曲）
  online_url  TEXT,                               -- 在线URL（仅在线歌曲）
  cover_url   TEXT,                               -- 封面URL（冗余存储）
  added_at    TEXT NOT NULL,                      -- 添加到歌单的时间

  FOREIGN KEY (playlist_id) REFERENCES playlists(id) ON DELETE CASCADE
);
```

**索引设计**：

```sql
-- 加速"查询某歌单所有歌曲"的操作
CREATE INDEX idx_ps_playlist_id ON playlist_songs(playlist_id);

-- 加速"某歌曲在哪些歌单中"的查询
CREATE INDEX idx_ps_song_id ON playlist_songs(song_id);

-- 唯一约束：同一首歌在同一个歌单中只能出现一次
CREATE UNIQUE INDEX idx_ps_playlist_song ON playlist_songs(playlist_id, song_id);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `INTEGER` | PK, AUTO | 自增主键 |
| `playlist_id` | `TEXT` | NOT NULL, FK | 关联歌单ID，级联删除 |
| `song_id` | `TEXT` | NOT NULL | 歌曲ID（来源标识） |
| `song_source` | `TEXT` | NOT NULL | `local` 或 `online` |
| `title` | `TEXT` | NOT NULL | 歌曲标题（冗余存储） |
| `artist` | `TEXT` | NOT NULL | 歌手名（冗余存储） |
| `duration` | `INTEGER` | 可空 | 时长，毫秒（冗余存储） |
| `file_path` | `TEXT` | 可空 | 本地文件路径（本地歌曲必有） |
| `online_url` | `TEXT` | 可空 | 在线播放URL（在线歌曲必有） |
| `cover_url` | `TEXT` | 可空 | 封面图URL（冗余存储） |
| `added_at` | `TEXT` | NOT NULL | 添加到歌单的时间 |

### 3.3 song_stats — 歌曲播放统计表

存储每首歌曲的播放统计汇总，支持两个统计维度：按歌曲本身 / 按歌曲分类。

```sql
CREATE TABLE song_stats (
  song_id         TEXT PRIMARY KEY,    -- 歌曲ID（本地为文件路径哈希）
  title           TEXT NOT NULL,       -- 歌曲标题（冗余）
  artist          TEXT,                -- 歌手（冗余）
  genre           TEXT,                -- 歌曲分类/风格（用户自定义，如：流行/摇滚/古典/电子/民谣/说唱）

  -- 基础统计
  play_count      INTEGER DEFAULT 0,   -- 累计播放次数
  total_seconds   INTEGER DEFAULT 0,   -- 累计播放时长（秒）
  last_played_at  TEXT,                -- 最后播放时间，ISO 8601 字符串

  -- 维度1：按歌曲本身的统计
  favorite_segment_start INTEGER,      -- 最受喜欢播放的片段开始时间（秒），如：120 表示 2:00 处
  favorite_segment_count INTEGER DEFAULT 0,  -- 该片段被重复播放的次数
  favorite_time_slot TEXT,             -- 主要播放时段：morning(6-12) / afternoon(12-18) / evening(18-24) / night(0-6)

  -- 时段播放次数分布（各字段互斥统计，各自在 0-24 范围内）
  morning_count     INTEGER DEFAULT 0,  -- 6:00-12:00 时段播放次数
  afternoon_count   INTEGER DEFAULT 0,  -- 12:00-18:00 时段播放次数
  evening_count     INTEGER DEFAULT 0,  -- 18:00-24:00 时段播放次数
  night_count       INTEGER DEFAULT 0,  -- 0:00-6:00 时段播放次数

  -- 维度2：按分类的统计（分类汇总表，非歌曲粒度）
  -- 注：分类统计不存在这里，而是通过 genre 字段 + 聚合查询实现

  created_at      TEXT NOT NULL,       -- 首次播放记录创建时间
  updated_at      TEXT NOT NULL        -- 最后更新时间
);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `song_id` | `TEXT` | PK | 歌曲唯一标识 |
| `title` | `TEXT` | NOT NULL | 歌曲标题（冗余存储，加速显示） |
| `artist` | `TEXT` | 可空 | 歌手名（冗余存储） |
| `genre` | `TEXT` | 可空 | 用户自定义分类，如"流行/摇滚/古典" |
| `play_count` | `INTEGER` | DEFAULT 0 | 累计播放次数 |
| `total_seconds` | `INTEGER` | DEFAULT 0 | 累计播放秒数 |
| `last_played_at` | `TEXT` | 可空 | 最后播放时间 |
| `favorite_segment_start` | `INTEGER` | 可空 | 最爱片段开始时间（秒），如 120 表示从 2:00 开始 |
| `favorite_segment_count` | `INTEGER` | DEFAULT 0 | 该片段被重复播放的次数 |
| `favorite_time_slot` | `TEXT` | 可空 | 主播放时段：morning / afternoon / evening / night |
| `morning_count` | `INTEGER` | DEFAULT 0 | 6:00-12:00 时段播放次数 |
| `afternoon_count` | `INTEGER` | DEFAULT 0 | 12:00-18:00 时段播放次数 |
| `evening_count` | `INTEGER` | DEFAULT 0 | 18:00-24:00 时段播放次数 |
| `night_count` | `INTEGER` | DEFAULT 0 | 0:00-6:00 时段播放次数 |
| `created_at` | `TEXT` | NOT NULL | 统计记录创建时间 |
| `updated_at` | `TEXT` | NOT NULL | 最后更新时间 |

**统计维度说明**：

1. **按歌曲本身统计**：
   - `favorite_time_slot`：统计播放时间分布在哪个时段，次数最多的时段记录为此值
   - `favorite_segment_start`：当用户拖动进度条回到某个位置超过 N 次（阈值可配置，如 3 次），记录该位置为最爱片段

2. **按分类统计**（聚合查询，不单独存表）：
   - 按 `genre` 分组，SUM(play_count)、SUM(total_seconds)
   - 按 `favorite_time_slot` 分组，COUNT 各类别歌曲的播放次数分布

记录用户播放过的歌曲历史，用于"最近播放"功能。

### 3.5 local_file_metadata — 本地音乐元数据表

存储本地音乐文件的扩展元数据，包括封面图路径、歌词文件路径、ID3 标签信息等，用于快速访问而无需重复解析音频文件。

```sql
CREATE TABLE local_file_metadata (
  song_id         TEXT PRIMARY KEY,    -- 歌曲ID（文件路径哈希）
  file_path       TEXT NOT NULL,       -- 原始音频文件路径
  cover_path      TEXT,                -- 封面图本地路径（从文件提取或网络下载）
  lyrics_path     TEXT,                -- 歌词文件（LRC）本地路径
  lyrics_raw      TEXT,                -- 内嵌歌词文本（纯文本，可选存储）
  album           TEXT,                -- 专辑名（从ID3提取）
  year            INTEGER,             -- 发行年份
  track_number    INTEGER,             -- 曲目编号
  disc_number     INTEGER,             -- 碟片编号
  composer        TEXT,                -- 作曲
  lyricist        TEXT,                 -- 作词
  is_cover_fetched INTEGER DEFAULT 0,   -- 封面是否已从网络补全：0=否，1=是
  is_lyrics_fetched INTEGER DEFAULT 0, -- 歌词是否已从网络补全：0=否，1=是
  created_at      TEXT NOT NULL,       -- 首次扫描时间
  updated_at      TEXT NOT NULL        -- 最后更新时间
);
```

**索引设计**：

```sql
-- 加速"查询未补全封面的歌曲"操作
CREATE INDEX idx_lfm_cover ON local_file_metadata(is_cover_fetched);

-- 加速"查询未补全歌词的歌曲"操作
CREATE INDEX idx_lfm_lyrics ON local_file_metadata(is_lyrics_fetched);
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `song_id` | `TEXT` | PK | 歌曲唯一标识 |
| `file_path` | `TEXT` | NOT NULL | 音频文件完整路径 |
| `cover_path` | `TEXT` | 可空 | 封面图本地路径 |
| `lyrics_path` | `TEXT` | 可空 | LRC 歌词文件本地路径 |
| `lyrics_raw` | `TEXT` | 可空 | 内嵌歌词原始文本 |
| `album` | `TEXT` | 可空 | 专辑名 |
| `year` | `INTEGER` | 可空 | 发行年份 |
| `track_number` | `INTEGER` | 可空 | 曲目编号 |
| `disc_number` | `INTEGER` | 可空 | 碟片编号 |
| `composer` | `TEXT` | 可空 | 作曲 |
| `lyricist` | `TEXT` | 可空 | 作词 |
| `is_cover_fetched` | `INTEGER` | DEFAULT 0 | 封面是否已从网络补全 |
| `is_lyrics_fetched` | `INTEGER` | DEFAULT 0 | 歌词是否已从网络补全 |
| `created_at` | `TEXT` | NOT NULL | 首次扫描时间 |
| `updated_at` | `TEXT` | NOT NULL | 最后更新时间 |

### 3.6 sync_queue — OSS 同步队列表

管理本地音乐文件与阿里云 OSS 之间的同步任务队列，记录待上传、待下载、失败、重试等状态的条目。

```sql
CREATE TABLE sync_queue (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增主键
  song_id       TEXT NOT NULL,                      -- 歌曲ID
  file_path     TEXT NOT NULL,                       -- 本地文件路径
  oss_key       TEXT NOT NULL,                       -- OSS 上的存储路径（Object Key）
  operation     TEXT NOT NULL,                       -- 操作类型：upload / download / delete
  status        TEXT NOT NULL,                       -- 状态：pending / in_progress / completed / failed
  priority      INTEGER DEFAULT 0,                   -- 优先级，数值越大优先级越高
  retry_count   INTEGER DEFAULT 0,                   -- 已重试次数
  max_retries   INTEGER DEFAULT 3,                   -- 最大重试次数
  error_message TEXT,                                -- 失败时的错误信息
  file_hash     TEXT,                                -- 文件 MD5 哈希（用于校验）
  file_size     INTEGER,                             -- 文件大小（字节）
  scheduled_at  TEXT,                                -- 计划执行时间，ISO 8601 字符串
  started_at    TEXT,                                -- 开始执行时间
  completed_at  TEXT,                                -- 完成时间
  created_at    TEXT NOT NULL                        -- 队列创建时间
);
```

**索引设计**：

```sql
-- 加速"查询待处理队列"的操作（按优先级倒序）
CREATE INDEX idx_sq_status_priority ON sync_queue(status, priority DESC);

-- 加速"查询某歌曲的同步状态"操作
CREATE INDEX idx_sq_song_id ON sync_queue(song_id);

-- 加速"查询重试队列"操作
CREATE INDEX idx_sq_failed ON sync_queue(status, retry_count) WHERE status = 'failed';
```

**字段说明**：

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `INTEGER` | PK, AUTO | 自增主键 |
| `song_id` | `TEXT` | NOT NULL | 歌曲ID |
| `file_path` | `TEXT` | NOT NULL | 本地文件路径 |
| `oss_key` | `TEXT` | NOT NULL | OSS Object Key |
| `operation` | `TEXT` | NOT NULL | `upload` / `download` / `delete` |
| `status` | `TEXT` | NOT NULL | `pending` / `in_progress` / `completed` / `failed` |
| `priority` | `INTEGER` | DEFAULT 0 | 优先级 |
| `retry_count` | `INTEGER` | DEFAULT 0 | 当前已重试次数 |
| `max_retries` | `INTEGER` | DEFAULT 3 | 最大允许重试次数 |
| `error_message` | `TEXT` | 可空 | 失败原因 |
| `file_hash` | `TEXT` | 可空 | MD5 哈希（用于完整性校验） |
| `file_size` | `INTEGER` | 可空 | 文件大小（字节） |
| `scheduled_at` | `TEXT` | 可空 | 计划执行时间 |
| `started_at` | `TEXT` | 可空 | 开始执行时间 |
| `completed_at` | `TEXT` | 可空 | 完成时间 |
| `created_at` | `TEXT` | NOT NULL | 队列条目创建时间 |

## 4. 数据库初始化SQL

完整建库脚本：

```sql
-- 创建 playlists 表
CREATE TABLE playlists (
  id          TEXT PRIMARY KEY,
  name        TEXT NOT NULL,
  cover_url   TEXT,
  description TEXT,
  creator     TEXT NOT NULL,
  type        TEXT NOT NULL,
  song_count  INTEGER DEFAULT 0,
  created_at  TEXT NOT NULL,
  updated_at  TEXT NOT NULL
);

CREATE INDEX idx_playlists_created_at ON playlists(created_at DESC);
CREATE INDEX idx_playlists_type ON playlists(type);

-- 创建 playlist_songs 表
CREATE TABLE playlist_songs (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  playlist_id TEXT NOT NULL,
  song_id     TEXT NOT NULL,
  song_source TEXT NOT NULL,
  title       TEXT NOT NULL,
  artist      TEXT NOT NULL,
  duration    INTEGER,
  file_path   TEXT,
  online_url  TEXT,
  cover_url   TEXT,
  added_at    TEXT NOT NULL,

  FOREIGN KEY (playlist_id) REFERENCES playlists(id) ON DELETE CASCADE
);

CREATE INDEX idx_ps_playlist_id ON playlist_songs(playlist_id);
CREATE INDEX idx_ps_song_id ON playlist_songs(song_id);
CREATE UNIQUE INDEX idx_ps_playlist_song ON playlist_songs(playlist_id, song_id);

-- 创建 song_stats 表
CREATE TABLE song_stats (
  song_id               TEXT PRIMARY KEY,
  title                 TEXT NOT NULL,
  artist                TEXT,
  genre                 TEXT,
  play_count            INTEGER DEFAULT 0,
  total_seconds         INTEGER DEFAULT 0,
  last_played_at        TEXT,
  favorite_segment_start INTEGER,
  favorite_segment_count INTEGER DEFAULT 0,
  favorite_time_slot    TEXT,
  morning_count         INTEGER DEFAULT 0,
  afternoon_count       INTEGER DEFAULT 0,
  evening_count         INTEGER DEFAULT 0,
  night_count           INTEGER DEFAULT 0,
  created_at            TEXT NOT NULL,
  updated_at            TEXT NOT NULL
);

-- 创建 local_file_metadata 表
CREATE TABLE local_file_metadata (
  song_id               TEXT PRIMARY KEY,
  file_path             TEXT NOT NULL,
  cover_path            TEXT,
  lyrics_path           TEXT,
  lyrics_raw            TEXT,
  album                 TEXT,
  year                  INTEGER,
  track_number          INTEGER,
  disc_number           INTEGER,
  composer              TEXT,
  lyricist              TEXT,
  is_cover_fetched      INTEGER DEFAULT 0,
  is_lyrics_fetched     INTEGER DEFAULT 0,
  created_at            TEXT NOT NULL,
  updated_at            TEXT NOT NULL
);

CREATE INDEX idx_lfm_cover ON local_file_metadata(is_cover_fetched);
CREATE INDEX idx_lfm_lyrics ON local_file_metadata(is_lyrics_fetched);

-- 创建 sync_queue 表
CREATE TABLE sync_queue (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id       TEXT NOT NULL,
  file_path     TEXT NOT NULL,
  oss_key       TEXT NOT NULL,
  operation     TEXT NOT NULL,
  status        TEXT NOT NULL,
  priority      INTEGER DEFAULT 0,
  retry_count   INTEGER DEFAULT 0,
  max_retries   INTEGER DEFAULT 3,
  error_message TEXT,
  file_hash     TEXT,
  file_size     INTEGER,
  scheduled_at  TEXT,
  started_at    TEXT,
  completed_at  TEXT,
  created_at    TEXT NOT NULL
);

CREATE INDEX idx_sq_status_priority ON sync_queue(status, priority DESC);
CREATE INDEX idx_sq_song_id ON sync_queue(song_id);

-- 创建 played_history 表
CREATE TABLE played_history (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id     TEXT NOT NULL,
  song_source TEXT NOT NULL,
  title       TEXT NOT NULL,
  artist      TEXT NOT NULL,
  duration    INTEGER,
  file_path   TEXT,
  online_url  TEXT,
  cover_url   TEXT,
  played_at   TEXT NOT NULL
);

CREATE INDEX idx_ph_played_at ON played_history(played_at DESC);
CREATE INDEX idx_ph_song_id ON played_history(song_id);
```

## 6. 版本升级策略

当数据库结构需要变更时（如新增表、字段、索引），在 `database_helper.dart` 的 `onUpgrade` 回调中处理：

```dart
onUpgrade: (db, oldVersion, newVersion) {
  // 迁移脚本
  if (oldVersion < 2) {
    db.execute('ALTER TABLE playlists ADD COLUMN tags TEXT');
  }
}
```
