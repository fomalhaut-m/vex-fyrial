# 播放统计实现方案设计

## 1. 统计数据结构

### 1.1 歌曲播放统计表

基于已设计的 `song_stats` 表扩展：

```sql
CREATE TABLE song_stats (
  id                       INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id                  TEXT NOT NULL UNIQUE,           -- 歌曲唯一 ID
  song_name                TEXT NOT NULL,
  artist                   TEXT,
  genre                    TEXT DEFAULT '未分类',          -- 用户手动分类标签

  play_count               INTEGER NOT NULL DEFAULT 0,    -- 播放次数
  total_seconds            INTEGER NOT NULL DEFAULT 0,    -- 累计播放时长（秒）

  last_played_at           INTEGER,                       -- 最后播放时间（Unix timestamp s）

  -- 时段统计
  morning_count            INTEGER NOT NULL DEFAULT 0,    -- 6:00-12:00 播放次数
  afternoon_count          INTEGER NOT NULL DEFAULT 0,    -- 12:00-18:00 播放次数
  evening_count            INTEGER NOT NULL DEFAULT 0,   -- 18:00-24:00 播放次数
  night_count              INTEGER NOT NULL DEFAULT 0,    -- 0:00-6:00 播放次数
  favorite_time_slot       TEXT,                          -- morning/afternoon/evening/night（播放次数最多的时段）

  -- 片段统计（30秒为一个片段）
  favorite_segment_start   INTEGER,                       -- 最爱片段起始秒数
  favorite_segment_count   INTEGER NOT NULL DEFAULT 0,    -- 该片段被 seek 到的累计次数

  -- 元数据
  created_at               INTEGER NOT NULL,
  updated_at               INTEGER NOT NULL
);

CREATE INDEX idx_genre ON song_stats(genre);
CREATE INDEX idx_last_played ON song_stats(last_played_at DESC);
```

## 2. 播放时段统计实现

### 2.1 时段划分

| 时段 | 时间范围 | 枚举值 |
|------|----------|--------|
| 早晨 | 06:00 – 12:00 | `morning` |
| 下午 | 12:00 – 18:00 | `afternoon` |
| 傍晚 | 18:00 – 24:00 | `evening` |
| 深夜 | 00:00 – 06:00 | `night` |

### 2.2 统计触发时机

每次播放开始时记录当前时段：

```dart
void onPlaybackStart(Song song) {
  final hour = DateTime.now().hour;
  final slot = _getTimeSlot(hour); // morning/afternoon/evening/night
  _pendingStats[song.id] ??= SongStats(songId: song.id);
  _pendingStats[song.id]!.incrementSlot(slot);
}

String _getTimeSlot(int hour) {
  if (hour >= 6 && hour < 12) return 'morning';
  if (hour >= 12 && hour < 18) return 'afternoon';
  if (hour >= 18 && hour < 24) return 'evening';
  return 'night';
}
```

### 2.3 favorite_time_slot 汇总

每次播放结束时，计算并更新 `favorite_time_slot`：

```dart
void _updateFavoriteTimeSlot(SongStats stats) {
  final slots = {
    'morning': stats.morningCount,
    'afternoon': stats.afternoonCount,
    'evening': stats.eveningCount,
    'night': stats.nightCount,
  };
  stats.favoriteTimeSlot = slots.entries
    .reduce((a, b) => a.value > b.value ? a : b)
    .key;
}
```

## 3. 最爱片段统计实现

### 3.1 片段长度与计数逻辑

- **片段长度**：30 秒
- **触发条件**：用户 seek 到某个位置超过 N 次（可配置，默认 N=3）
- **记录方式**：统计每个 30 秒区间被 seek 的次数

```dart
class SegmentTracker {
  final Map<int, int> segmentHits = {}; // {segmentIndex: hitCount}

  void onSeek(int positionSeconds) {
    final segmentIndex = positionSeconds ~/ 30;
    segmentHits[segmentIndex] = (segmentHits[segmentIndex] ?? 0) + 1;
  }

  // 找出被访问次数最多的片段
  (int startSecond, int count)? findFavoriteSegment() {
    if (segmentHits.isEmpty) return null;
    final sorted = segmentHits.entries.toList()
      ..sort((a, b) => b.value.compareTo(a.value));
    final top = sorted.first;
    if (top.value < 3) return null; // 未达阈值
    return (top.key * 30, top.value);
  }
}
```

### 3.2 数据更新

播放结束时（播放时长 ≥ 60 秒才计入，避免误触发）：

```dart
void onPlaybackEnd(Song song, int durationSeconds) {
  if (durationSeconds < 60) return;

  final stats = _getOrCreateStats(song.id);
  final segment = segmentTrackers[song.id]?.findFavoriteSegment();
  if (segment != null) {
    stats.favoriteSegmentStart = segment.startSecond;
    stats.favoriteSegmentCount = segment.count;
  }
  // 重置本轮片段追踪
  segmentTrackers[song.id]?.reset();
}
```

## 4. 按分类统计实现

### 4.1 分类来源

- **用户手动标签**：用户在歌曲详情页/列表管理中为歌曲打标签
- **默认分类**：未分类的歌曲归为 `未分类`

```sql
-- 用户标签表（扩展 song_genres 表）
CREATE TABLE song_genres (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  song_id   TEXT NOT NULL UNIQUE,
  genre     TEXT NOT NULL DEFAULT '未分类',
  updated_at INTEGER NOT NULL
);
```

用户可自定义分类：流行 / 摇滚 / 古典 / 电子 / 民谣 / 说唱 / 爵士 / 蓝调 / 其他

### 4.2 聚合查询

```sql
-- 按分类聚合统计
SELECT
  genre,
  COUNT(*) as song_count,
  SUM(play_count) as total_plays,
  SUM(total_seconds) as total_seconds,
  AVG(play_count) as avg_plays,
  MAX(last_played_at) as last_played
FROM song_stats
WHERE genre IS NOT NULL
GROUP BY genre
ORDER BY total_plays DESC;

-- 按时段分布（嵌套查询）
SELECT
  genre,
  SUM(morning_count)    as morning_plays,
  SUM(afternoon_count)  as afternoon_plays,
  SUM(evening_count)    as evening_plays,
  SUM(night_count)      as night_plays
FROM song_stats
GROUP BY genre;
```

## 5. 性能优化

### 5.1 异步写入

统计更新在播放线程异步执行，不阻塞音频播放：

```dart
// 使用 isolate 或队列处理统计写入
final _statsQueue = Queue<StatsUpdate>();

void onPlaybackEvent(PlaybackEvent event) {
  _statsQueue.add(StatsUpdate.fromEvent(event));
  // 主线程立即返回，不等待写入
}
```

后台 Worker 每 60 秒批量处理队列：

```dart
void _flushStatsPeriodically() async {
  while (true) {
    await Future.delayed(Duration(seconds: 60));
    final batch = _statsQueue.takeBatch(100); // 最多一次处理100条
    await _batchUpdateToDb(batch);
  }
}
```

### 5.2 内存缓存策略

| 策略 | 说明 |
|------|------|
| 启动时加载 | App 启动时将 `song_stats` 表加载到内存 Map |
| 实时更新缓存 | 播放时更新内存缓存 |
| 退出时写库 | App 退出时调用 `_flushAllStats()` 确保全部落盘 |
| 增量更新 | 写库时只更新变更字段（PlayCount、totalSeconds 等），减少 SQL 执行量 |

```dart
class StatsCache {
  final Map<String, SongStats> _cache = {};

  void loadFromDb() {
    final rows = await db.query('song_stats');
    for (final row in rows) {
      _cache[row['song_id']] = SongStats.fromRow(row);
    }
  }

  void incrementPlayCount(String songId) {
    _cache[songId]?.playCount++;
  }

  Future<void> flushAll() async {
    for (final stats in _cache.values) {
      await _updateDb(stats);
    }
  }
}
```

## 7. 数据完整性保障

- **首次播放创建记录**：歌曲首次被播放时自动创建 `song_stats` 记录
- **播放时间 ≥ 10 秒才计入**：避免快进误统计
- **崩溃恢复**：内存中未落盘的统计，App 退出时写入；重启后检查未flush的记录并补充
- **迁移脚本**：分类字段默认为"未分类"，graceful 处理旧数据
