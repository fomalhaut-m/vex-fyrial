## 二、可测试性分析

### 2.1 PlayerNotifier

| 方法 | 可测试性 | 说明 |
|------|----------|------|
| `playSong()` | ✅ | 可以 mock AudioPlayer |
| `togglePlayPause()` | ✅ | 纯逻辑，无外部依赖 |
| `setPlayMode()` | ✅ | 纯逻辑 |
| `seekTo()` | ✅ | 可以 mock AudioPlayer |
| `setPlaylist()` | ✅ | 可以 mock AudioPlayer |

**建议**：
- `AudioPlayer` 是直接依赖，建议通过接口解耦（`IPlayer`）
- 可以对 `PlayerState.progressPercent` 进行边界测试（除零保护）

### 2.2 LocalMusicNotifier

| 方法 | 可测试性 | 说明 |
|------|----------|------|
| `scanAllMusic()` | ⚠️ | 依赖文件系统 |
| `loadAllSongs()` | ✅ | 可以 mock Database |
| `searchSongs()` | ✅ | 可以 mock Database |
| `deleteSong()` | ✅ | 可以 mock Database |

**建议**：
- `scanAllMusic()` 需要真实文件系统或 mock `Directory.list()`
- 可以对 `_isSupportedFormat()` 进行单元测试（各种扩展名）

### 2.3 HealthCheckService

| 方法 | 可测试性 | 说明 |
|------|----------|------|
| `runAllChecks()` | ✅ | 返回 bool |
| `getFriendlyMessage()` | ✅ | 可以 assert 输出 |

**建议**：
- 可以对 `_parseLinuxMemory()` / `_parseMacOSMemory()` 进行单元测试
- 可以对 `getFailedFeatures()` / `getWarningFeatures()` 进行输出验证
