## 一、项目概览

### 1.1 代码结构
- **Dart 文件数**：22个
- **模块数**：4个主Tab + 全局组件
- **技术栈**：Flutter + Riverpod + just_audio + sqflite + go_router + audio_service

### 1.2 目录结构
```
app/lib/
├── main.dart                      # 入口（健康检测、异常处理）
├── app/
│   ├── core/theme.dart            # 主题配置
│   ├── modules/
│   │   ├── home/home_page.dart    # 首页（4 Tab导航）
│   │   ├── player/                # Tab1：播放器
│   │   │   ├── player.dart
│   │   │   ├── player_page.dart   # 全屏播放页
│   │   │   └── player_tab.dart    # Tab内容
│   │   ├── playlist/              # Tab2：播放列表
│   │   ├── stats/                # Tab3：统计
│   │   └── settings/             # Tab4：设置
│   └── routes/app_router.dart    # 路由配置
├── data/
│   ├── database/
│   │   ├── database_helper.dart   # 数据库管理
│   │   └── tables.dart           # 表名常量
│   └── models/
│       └── song_model.dart       # 歌曲模型
├── providers/
│   ├── player_provider.dart      # 播放器状态
│   ├── local_music_provider.dart # 本地音乐扫描
│   └── tab_index_provider.dart   # Tab索引
├── services/
│   ├── audio_handler_service.dart # 后台播放+通知栏
│   └── health_check_service.dart  # 12项健康检测
├── widgets/mini_player.dart      # 全局迷你播放器
└── router/app_router.dart        # go_router配置
```

### 1.3 核心模块
| 模块 | 职责 | 关键文件 |
|------|------|----------|
| PlayerModule | 音频播放控制、队列管理、歌词同步 | player_provider.dart, audio_handler_service.dart |
| LocalMusicModule | 本地音乐扫描、元数据解析 | local_music_provider.dart |
| PlaylistModule | 歌单管理、收藏、最近播放 | playlist_tab.dart |
| StatsModule | 播放统计聚合 | stats_tab.dart |
| SettingsModule | OSS配置、主题、缓存 | settings_tab.dart |
| MiniPlayer | 全局常驻迷你播放器 | mini_player.dart |

### 2.1 第一遍：按模块审查（9轮）

> 目标：验证每个模块的功能完整性、代码正确性、与设计的对齐

## 四、审查标准

### 4.1 代码质量标准（开发准则要求）
- [ ] 中文注释完整，关键逻辑有解释
- [ ] 命名清晰（变量/函数/类名一看就懂）
- [ ] 目录结构符合规范
- [ ] 无性能问题（内存泄漏、流未取消）
- [ ] 功能实现与UI设计一致

### 4.2 PRD对齐标准
- [ ] P0核心功能（流畅播放+后台保活+低内存）完整实现
- [ ] P1/P2功能按优先级实现
- [ ] 与UI设计文档一致

### 4.3 架构标准
- [ ] 分层清晰（表现层/状态层/业务层/数据层）
- [ ] 模块间依赖正确，无循环依赖
- [ ] 异常兜底完整
- [ ] 资源正确释放
