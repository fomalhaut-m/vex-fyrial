# 设计愿景

## 产品定位

**Vexfy** — 轻量、快速、一体化的音乐伴侣

不做臃肿的音乐航母，而是专注于 **播放体验** 和 **本地+在线一体化** 的轻量播放器。

## 设计理念

### 1. 沉浸优先

深色背景让内容突出，专辑封面成为视觉焦点，用户聚焦于音乐本身。

- **全沉浸体验**：状态栏、内容区域、封面无缝融合
- **封面优先**：封面占据视觉重心，60%+ 的视觉面积
- **内容呼吸感**：大量留白，密集信息区有明确的视觉边界

### 2. 简洁克制

- 去掉多余装饰：无需花哨的渐变、阴影、边框
- 信息层级清晰：主信息 > 次要信息 > 辅助信息，三级分明
- 操作路径短：3步内完成任何操作（打开App → 找到内容 → 播放）
- 功能精而非多：只做核心功能，每个功能做到极致

### 3. 流畅动效

- 页面切换流畅（300ms ease-in-out）
- 歌词滚动自然跟随（同步播放位置，视觉平滑过渡）
- 迷你播放器常驻不打断体验（随时可见播放状态）
- 操作反馈即时（点击、滑动、拖拽都有明确反馈）
- 封面旋转（播放时慢速旋转，营造音乐生命力）

### 4. 一体化体验

本地音乐和在线音乐在同一个界面融合，用户无需切换 App。

- **统一的视觉语言**：本地和在线内容使用相同的卡片、列表、播放控件
- **统一的交互体验**：搜索、播放、收藏等操作无差异化
- **数据互通**：收藏的歌单、本地歌曲、播放历史统一管理

## 视觉关键词

| 关键词 | 解释 | 设计表现 |
|--------|------|----------|
| **深邃** | 深黑色背景 + 绿色点缀，营造音乐厅氛围 | #121212 背景，#1DB954 品牌色 |
| **呼吸感** | 大量留白，内容不过分密集 | 卡片间距 12dp，页面边距 16dp |
| **可点击感** | 按钮/卡片有明确的点击反馈 | 按压态 opacity 0.7，hover 态 scale 1.02 |
| **一致性** | 全局统一的圆角、间距、阴影规范 | 圆角 4/8/12dp，阴影统一，间距 4/8/12/16/24dp |
| **流动感** | 音乐是流动的，界面也应有这种质感 | 进度条平滑动画，歌词无缝滚动 |

### 我们的差异化定位

| 维度 | 描述 |
|------|------|
| **更轻量** | 不做社交、评论、社区，只做音乐播放 |

## 设计语言

### 色彩

- **主色调**：Spotify 绿（#1DB954）— 代表音乐、生命、流动
- **背景**：深黑渐变（#121212 → #181818）— 沉浸感
- **卡片**：略浅于背景（#181818）— 有层次但不突兀
- **高亮**：纯白（#FFFFFF）— 当前行歌词、重要文字
- **次要文字**：灰白（#B3B3B3）— 歌手、时长等次要信息

详见：[color-theme.md](./color-theme.md)

### 字体

- **中文**：PingFang SC（iOS），Noto Sans SC（Android）
- **英文/数字**：SF Pro Display（iOS），Roboto（Android）
- **字号层级**：标题 20-28px，正文 14-16px，辅助 10-12px

详见：[typography.md](./typography.md)

### 布局

- **页面结构**：顶部导航 + 内容 + 底部 TabBar + 迷你播放器
- **安全区**：适配刘海屏、Home 键区域
- **间距系统**：4dp 基础单位，4/8/12/16/24/32dp 间距规范
- **圆角系统**：4dp（小）/ 8dp（中）/ 12dp（大）/ 16dp（卡片）

详见：[layout-structure.md](./layout-structure.md)

### 动效

- **时长**：标准 300ms，快速反馈 150ms，复杂动画 400ms
- **缓动函数**：ease-in-out（默认），ease-out（出现），ease-in（消失）
- **页面切换**：水平滑动 300ms
- **全屏播放页**：从底部滑入 iOS 风格

详见：[layout-structure.md](./layout-structure.md)

## 设计目标

### 核心目标

1. **让音乐成为主角**：界面不抢戏，内容（封面、歌词、歌曲信息）是最重要的
2. **3步完成任何操作**：从打开 App 到播放歌曲，最多 3 步
3. **60fps 流畅体验**：滑动、切换、动画无卡顿

### 非目标（明确不做）

- 评论功能
- 社交动态
- 直播
- 商城
- 复杂的账户体系

## 附录：设计价值观

> **好的设计是不被注意到的设计。**
> 
> 当用户沉浸在音乐中时，不会意识到界面的存在。
> 当用户需要操作时，界面响应即时且直观。
> 当用户不使用时，界面安静守候。
