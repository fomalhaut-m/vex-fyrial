# 产品需求文档 (PRD) - v3（核心优先级修订）

> ⚠️ v3 修订核心：流畅播放 + 后台保活 + 低内存占用 = 真正核心；统计/歌词补全 = 锦上添花

---

## 1. 产品概述

**产品名称**：Vexfy
**产品类型**：移动端个人音乐播放器
**核心价值**：流畅、稳定的本地音乐播放 + OSS云端同步，打造轻量高效的个人音乐库
**目标平台**：Android / iOS

**产品愿景**：做一个**流畅不卡、后台稳定、内存友好**的个人音乐播放器。音乐来自本地文件和私有OSS存储，不依赖任何在线音乐平台。

---

## 2. 用户核心诉求（v3 重点）

> Luke 明确：统计不是核心，流畅播放 + 后台保活 + 低内存才是核心

### 核心优先级（必须做好）
1. **流畅播放** — 不卡顿、不crash、响应快
2. **后台保活** — 切App/锁屏继续播、通知栏控制
3. **内存占用低** — 不占系统资源，续航友好

### 锦上添花（可选）
- 播放统计（次数/时长）
- 封面歌词自动补全
- 歌曲分类/时段统计

## 3. 核心功能优先级

### P0（必须）- 核心中的核心
- 本地音乐文件扫描（指定目录递归扫描）
- 音频播放（播放/暂停/切歌/进度拖动）— **流畅不卡**
- 后台播放 + 通知栏控制 — **切App/锁屏继续播**
- 内存占用低 — 不杀后台、不占系统资源

### P1（重要）- 核心体验
- OSS 连接配置（AccessKey/Secret/Bucket/Endpoint）
- 本地 ↔ OSS 双向同步（上传/下载/增量同步）
- 底部迷你播放器（全局常驻）
- 全屏播放页（封面+播控）
- 歌词滚动同步（LRC 解析 + 实时滚动）

### P2（增强）- 功能完善
- 封面自动补全（联网搜索，嵌入 ID3 标签）
- 歌词自动补全（LRC 搜索）
- 本地歌单创建/编辑/删除（SQLite）
- 歌曲搜索（按歌名/歌手/专辑）
- 播放模式（列表循环/单曲循环/随机）
- 音质/倍速设置

### P3（可选）- 增值功能
- 播放统计（每首歌播放次数/总时长/最后播放时间）
- 按分类统计（流行/摇滚/古典等）
- 按播放时段统计（morning/afternoon/evening/night）
- 缓存清理

## 5. 核心场景

### 场景 1：本地音乐播放（核心）
打开App → 扫描本地音乐 → 显示歌曲列表 → 点击播放 → 流畅播放不卡

> **核心需求**：播放流畅、响应快、不crash

### 场景 2：后台保活（核心）
切到微信/导航/其他App → 音乐继续播放 → 锁屏 → 通知栏控制上一首/暂停/下一首

> **核心需求**：后台稳定不掉线、通知栏交互完整

### 场景 3：OSS 双向同步
在其他设备上传音乐到 OSS → App 检测到变化 → 自动同步到本地
用户在 App 内删除/修改音乐 → 同步到 OSS

> **需求**：双向同步稳定、可配置同步目录

### 场景 4：歌词滚动
播放歌曲 → 显示歌词视图 → 歌词随播放滚动 → 当前行高亮

> **需求**：歌词滚动流畅、精准

### 场景 5：歌单管理
创建个人歌单 → 添加本地歌曲 → 歌单存储 SQLite

## 6. KPI / 衡量指标

- **首次播放成功率 > 98%**（流畅不卡）
- **后台保活率 > 95%**（切App/锁屏不掉线）
- **内存占用 < 150MB**（轻量不占资源）
- 本地扫描成功率 > 99%
- 页面流畅度 60fps

## 7. 竞品定位

| 竞品 | 痛点 | 我们解决什么 |
|------|------|------------|
| 网易云音乐 | 太重、推荐烦人 | 纯本地、轻量、无推荐 |
| QQ音乐 | 版权复杂、太贵 | 纯本地，不需要版权 |
| Poweramp | 无云同步 | OSS双向同步 |
| 国内ROM后台 | 系统杀进程 | 专门的后台保活方案 |

**核心差异化**：流畅 + 轻量 + 私有OSS同步

---

## 8. 技术约束

- **OSS**：阿里云 OSS（用户已有阿里云账号）
- **音乐格式**：MP3、AAC、FLAC、WAV、MP4
- **元数据**：MP3 使用 ID3v2 标签
- **存储**：本地音乐路径 + SQLite（歌单/配置）
- **离线**：完全支持，所有功能不依赖网络（OSS同步除外）
