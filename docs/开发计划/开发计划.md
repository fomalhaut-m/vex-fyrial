# Flutter 音乐 APP - 详细开发计划

## 一、项目定位

**项目名称**：Vexfy
**工作目录**：/home/luke/work/M-stand/
**架构模式**：分层架构 + 单向数据流
**开发模式**：先单点突破，再聚合集成

---

## 二、核心技术栈

| 模块 | 技术方案 |
|------|---------|
| 状态管理 | GetX（全局状态 + 依赖注入） |
| 本地数据库 | sqflite + path_provider |
| 网络请求 | dio + dio_cache |
| 播放器核心 | just_audio |
| 后台播放 | audio_service + just_audio |
| 本地音乐扫描 | on_audio_query |
| 路由管理 | GetX 路由（GetMaterialApp） |
| 缓存管理 | flutter_cache_manager |

## 四、页面结构

```

    ↓
引导页 (Guide) → 首次安装显示
    ↓
主页 (Home)
├── 推荐页 (Recommend)      ← Tab
├── 排行榜 (Ranking)        ← Tab
```

## 五、详细开发顺序（最小单元 → 聚合）

### 【阶段 0】工程骨架
**目标**：跑通项目结构，全功能空框架

**步骤**：
1. `flutter create vexfy --org com.example`
2. 配置 pubspec.yaml（所有依赖）
3. 创建目录结构
4. 配置 GetX 路由（所有页面注册）
5. 配置主题（配色、字体）
6. 创建骨架 Page（只有 Container 和 AppBar）
7. 验证：flutter run 能跑起来，所有页面能跳转

**交付物**：空壳 App，底部 Tab 能切换

---

### 【阶段 1】播放器核心（最小单元）
**目标**：最基础的音频播放功能

**步骤**：
1. 创建 `PlayerController`（GetX）
2. 集成 just_audio，初始化 `AudioPlayer` 实例
3. 实现 play() / pause() / stop() / seek()
4. 监听 position 流和 duration 流
5. 创建 3 个基础变量：`currentPosition`、`duration`、`isPlaying`
6. 添加上一首/下一首逻辑（hardcode 假数据）

**UI 交付**：
- 临时 `TestPlayerPage`：一个播放按钮、一个进度条、一个暂停按钮
- 点击播放 → 控制音频输出
- 进度条实时更新

**验证**：点击能播放内置测试音频，能暂停，能拖动进度条

---

### 【阶段 2】歌曲模型 + 数据层（最小单元）
**目标**：建立数据模型，理解数据流

**步骤**：
1. 创建 `SongModel`：id, title, artist, album, duration, url, coverUrl, lrcUrl
2. 创建 `PlaylistModel`：id, name, creator, songs[], coverUrl
3. 创建假数据 `FakeData`：5 首假歌曲，2 个假歌单
4. 在 `PlayerController` 添加 `currentSong` 和 `songList`
5. 实现切歌：点击列表中某首 → 播放这首

**UI 交付**：
- `SongTile` widget：封面缩略图 + 歌名 + 歌手 + 时长
- 临时列表页：显示假数据歌单

**验证**：能显示歌曲列表，点击某首开始播放，切歌时UI更新

---

### 【阶段 3】歌词解析（最小单元）
**目标**：LRC 文件解析 + 滚动

**步骤**：
1. 创建 `LrcModel`：time（秒）, text（歌词文本）
2. 创建 `LrcParser`：输入 LRC 文本字符串，输出 List<LrcModel>
3. 正则解析 `[mm:ss.xx]` 时间戳
4. 创建 `LrcView` widget：
   - 用 ListView 显示所有歌词行
   - 根据当前播放进度，计算当前行索引
   - 用 `AnimatedList` 或手动 `ScrollController` 滚动到当前行
   - 当前行高亮（变大/变绿）

**UI 交付**：
- `LrcView` widget：半透明黑色背景，歌词居中
- 当前行：白色大字；其他行：灰色小字

**验证**：播放时歌词自动滚动，手动拖动进度条歌词同步跳

---

### 【阶段 4】全屏播放器页面（聚合 1）
**目标**：整合阶段 1~3，做出一个像样的播放界面

**步骤**：
1. 创建 `PlayerPage`（全屏路由 `/player`）
2. 顶部：返回按钮 + 歌曲名 + 歌手名
3. 中部：专辑封面（大图，圆角，阴影）+ 歌词区域（二选一：封面优先 or 歌词优先，用 Tab 切换）
4. 底部：进度条（可拖动）+ 时间显示 + 上一首/播放暂停/下一首 + 收藏按钮 + 更多按钮
5. 底部工具栏：播放模式（循环/随机）+ 音质选择 + 倍速

**UI 交付**：
- 完整的 `PlayerPage`，可以叫出全屏播放
- 封面旋转动画（可选）
- 进度条拖动时时间跟着变

**验证**：从列表点一首歌 → 跳转全屏播放页 → 能完整播放、暂停、切歌、看歌词

---

### 【阶段 5】底部常驻迷你播放器（聚合 2）
**目标**：全局显示，不受路由影响

**步骤**：
1. 在 `HomePage` 的 Scaffold 底部放一个 `MiniPlayer` widget
2. 用 `GetX` 的 `Get.put()` 全局创建 `PlayerController`（不在 Page 里创建）
3. `MiniPlayer` 监听 `PlayerController` 的状态
4. 点击 `MiniPlayer` → 跳转全屏 `PlayerPage`
5. `HomePage` 的 body 用 `SafeArea` 留出底部迷你播放器的高度

**UI 交付**：
- 底部 64dp 高度：左侧封面 + 中间歌名/歌手 + 右侧播放/暂停按钮
- 进度条极简显示（一条细线）

**验证**：在 Home 的任意 Tab 下，底部都有迷你播放器；切歌时迷你播放器信息更新；点击跳转全屏

---

### 【阶段 6】本地音乐扫描（最小单元）
**目标**：读取本机音乐文件

**步骤**：
1. 集成 `on_audio_query`
2. 创建 `LocalMusicController`（GetX）
3. 调用 `onAudioQuery.querySongs()` 获取所有本地歌曲
4. 映射到 `SongModel`（有些字段可能为空，做兼容）
5. 创建 `LocalMusicPage`：列表显示 + 歌手分组 + 时长排序选项

**UI 交付**：
- `LocalMusicPage`：类似网易云音乐本地音乐 Tab
- 显示：歌曲封面（如果没有用默认图）+ 歌名 + 歌手 + 专辑 + 时长

**验证**：能列出真机/模拟器上真实的本地音乐

---

### 【阶段 7】网络请求层（最小单元）
**目标**：接入真实音乐 API

**步骤**：
1. 集成 `dio`，创建 `ApiService`
2. 找一个免费音乐 API（如 网易云音乐 API / QQ 音乐 API / 野月 API）
3. 实现接口：
   - 推荐歌单 `/recommend`
   - 排行榜 `/rank`
   - 歌单广场 `/square`
   - 搜索 `/search?keyword=`
   - 获取歌曲URL `/song/url?id=`
4. 创建 `MusicRepository` 封装 API 调用
5. 用假数据做 mock，等 API 通了换真数据

**UI 交付**：
- `RecommendPage`：横向滑动 banner + 推荐歌单列表
- `RankingPage`：排行榜列表（第一名大卡片，其余小卡片）
- `SquarePage`：歌单网格（2列）

**验证**：API 返回数据，页面能显示歌单列表

---

### 【阶段 8】搜索功能（最小单元）
**目标**：搜索歌曲、歌手、专辑

**步骤**：
1. 创建 `SearchController`（GetX）
2. 输入框 + 实时搜索（debounce 300ms）
3. 搜索结果分类显示：歌曲 / 歌手 / 专辑
4. 点击搜索结果 → 播放歌曲 或 进入歌手/专辑详情页

**UI 交付**：
- `SearchPage`：搜索框 + 历史记录 + 热搜榜 + 搜索结果列表
- `SongTile` 通用组件复用

**验证**：输入关键词 → 显示搜索结果 → 点击播放

---

### 【阶段 9】歌单功能（最小单元）
**目标**：创建、收藏、编辑歌单

**步骤**：
1. 集成 `sqflite`，创建数据库 `music.db`
2. 表设计：
   - `playlists`: id, name, coverUrl, createTime, updateTime
   - `playlist_songs`: playlistId, songId, orderIndex
3. 实现 `PlaylistService`：
   - `createPlaylist(name)`
   - `addSongToPlaylist(playlistId, songId)`
   - `removeSongFromPlaylist(playlistId, songId)`
   - `deletePlaylist(playlistId)`
   - `getUserPlaylists()`
4. 创建 `PlaylistController`（GetX）管理歌单状态

**UI 交付**：
- `MyPlaylistsPage`：我的歌单列表（网格/列表切换）
- `PlaylistDetailPage`：歌单内歌曲列表 + 播放全部按钮
- 收藏按钮：点击收藏到某个歌单（弹出歌单选择器）

**验证**：能创建新歌单、能添加歌曲、能查看歌单详情

---

### 【阶段 10】后台播放 + 通知栏（聚合 3）
**目标**：切到后台/锁屏时继续播放

**步骤**：
1. 集成 `audio_service`
2. 创建 `AudioHandlerService` 继承 `BaseAudioHandler`

4. 配置 `iOS Info.plist`：添加音频后台模式
5. 通知栏：显示歌曲名 + 歌手 + 封面图 + 播放/暂停/上一首/下一首按钮
6. 耳机线控：按下暂停/继续/切歌

**UI 交付**：
- 锁屏时通知栏有播控按钮
- 按耳机线控能切歌

**验证**：按 Home 键切到后台 → 音乐继续播放 → 通知栏有控制选项

**UI 交付**：

- `GuidePage`：3 张引导图 + 跳过按钮 + 立即体验按钮

## 七、核心原则

1. **先跑通，再丰富**：每个最小单元完成后都能独立运行
2. **数据流单向**：UI 只从 Controller 读取状态，Controller 操作 Model
3. **不抄作业**：播放器用 just_audio + audio_service，不依赖第三方封装好的播放器插件
4. **渐进式**：先假数据，再接 API；先单歌曲，再列表；先基础播放，再高级功能
5. **每步可验证**：每个阶段完成后都能 `flutter run` 看到效果
